1. Executive Overview

What Is It?

GCP Security Analyzer is a purpose-built command-line tool that connects directly to Google Cloud's Security Command Center (SCC), fetches active security findings, and transforms raw data into prioritized, actionable intelligence — complete with risk scores, compliance mapping, and executable remediation scripts.

The Problem vs. The Solution

Today (Manual Process)

  • Manually export CSV from SCC console
  • Risk prioritization is subjective and inconsistent
  • Remediation guidance is generic
  • Compliance mapping requires cross-referencing docs
  • Reports are static spreadsheets
  • Every report overwrites the last

With GCP Security Analyzer

  • One command fetches live findings via API
  • Deterministic 0–100 scoring with full transparency
  • Finding-specific bash/python scripts ready to execute
  • Automatic detection across 7 compliance frameworks
  • Interactive HTML dashboards + MD + JSON + CSV
  • Timestamped unique filenames for audit trail

3. Risk Scoring Methodology

Each finding is scored on a 0–100 scale using six weighted components:

ComponentMax PointsHow It Works
Base Severity40CRITICAL=40, HIGH=30, MEDIUM=20, LOW=10
CVSS Score30CVSS v3 base score × 3
Exploitability20In-the-wild (+10), Zero-day (+8), Activity level (+2–6), Has CVE (+2)
Finding Class10THREAT=10, VULNERABILITY=7, MISCONFIG=5, OBSERVATION=2
Resource Exposure10Public IP (+5), Internet-facing (+3), Critical resource (+2)
Compliance Impact10Has frameworks (+5), Detailed compliance (+3), Audit category (+2)

A category weight multiplier (0.8x–1.2x) is applied. A severity floor ensures GCP's CRITICAL (≥75) and HIGH (≥55) designations are never downgraded.

4. Frequently Asked Questions

General

What is GCP Security Analyzer? ▼
A command-line tool written in Go that fetches security findings from Google Cloud's Security Command Center, scores them using a multi-factor risk algorithm, maps them to compliance frameworks, generates remediation guidance, and produces interactive reports with executable fix scripts.
Why not just use the GCP Security Command Center console directly? ▼
SCC provides raw findings but lacks quantitative risk scoring, cross-framework compliance mapping, executable remediation scripts, and customizable multi-format reporting. This tool adds those layers of intelligence on top of SCC data.
Does this replace Security Command Center? ▼
No. It complements SCC by consuming its findings and adding risk scoring, prioritization, compliance mapping, and automated remediation that SCC does not provide natively.

Risk Scoring

How are findings scored? ▼
Each finding is scored on a 0–100 scale using six weighted components: Base Severity (40 pts), CVSS Score (30 pts), Exploitability (20 pts), Finding Class (10 pts), Resource Exposure (10 pts), and Compliance Impact (10 pts). A category weight multiplier (0.8x–1.2x) is applied to the raw sum, and a severity floor ensures GCP's own CRITICAL and HIGH designations are never downgraded.
Can the scoring weights be customized? ▼
Yes. The scoring engine accepts a configuration struct that can be tuned. Future versions will support YAML-based configuration files for easy customization.
Is the scoring methodology transparent? ▼
Fully. Every finding's risk score includes a component breakdown and a human-readable rationale string explaining exactly how the score was computed, including whether a severity floor was applied. No black boxes.

Remediation

What findings get remediation scripts? ▼
All CRITICAL-priority findings receive executable remediation scripts (bash or python). All findings — regardless of priority — receive remediation guidance including summary, next steps, estimated effort, and automation potential.
Are the scripts safe to run? ▼
Every script supports a DRY_RUN=true mode that previews actions without making changes. Scripts use set -euo pipefail for error handling and include detailed inline comments. Always review scripts before executing in production.
What categories have specialized scripts? ▼
11 categories: VPC Flow Logs, Private Google Access, Bucket Logging, Audit Logging, Firewall Rule Logging, Open Firewalls, Weak SSL/TLS, Service Account Keys, IAM Anomalies, Container Vulnerabilities, and OS Vulnerabilities. All other categories receive a generic inspection script.

Compliance

Which compliance frameworks are supported? ▼
CIS Benchmarks, PCI-DSS, HIPAA, SOC 2, ISO 27001, NIST 800-53, and GDPR. The tool parses compliance data embedded in SCC findings and maps them to specific framework controls.
Does this tool certify compliance? ▼
No. It identifies and maps compliance-relevant findings to framework controls. It does not perform audits or grant certifications. Use it as evidence during audit preparation.

Data & Security

Does any data leave our environment? ▼
By default, no. All processing happens locally on your workstation. The only exception is the optional --ai-enhance flag, which sends CRITICAL finding summaries to the Claude API. This requires explicit opt-in and an API key.
Are credentials stored by the tool? ▼
No. The tool uses GCP's standard Application Default Credentials (ADC) mechanism. No credentials are stored, logged, or transmitted by the tool itself.

Operations

Can this be integrated into CI/CD? ▼
Yes. The tool is a single binary with standard exit codes (0 = success, 1 = error). It can run in any CI/CD pipeline. The fetch command pulls live data without manual CSV exports.
How often should we run it? ▼
Recommended: daily for CRITICAL/HIGH monitoring, weekly for full analysis. Timestamped output files create a natural audit trail without manual file management.
What are the infrastructure requirements? ▼
A single Go binary (~15MB). No database, no containers, no external services required. Runs on macOS, Linux, and Windows.
Does it support multiple GCP organizations? ▼
Yes. Pass different --org-id values to the fetch command. Each run produces independently named report files.

AI Enhancement

What does --ai-enhance do? ▼
It sends CRITICAL finding details to the Claude API (Anthropic) for context-aware analysis. The AI provides deeper remediation recommendations, explains potential blast radius, and suggests related findings to investigate.
Is --ai-enhance required? ▼
No. It is entirely optional and requires an ANTHROPIC_API_KEY environment variable. The tool is fully functional without it.
